**Step 2 ‚Äî Add getRecentIds and storeRecentIds helpers to app.js**

Place these two functions immediately before the `workoutDays` constant declaration (after the pickRandom function, before the workoutDays comment block):

```javascript
// =============================================================================
// LOCALSTORAGE RECENCY HELPERS (private ‚Äî not exported)
// =============================================================================

/**
 * Read recent exercise IDs for a routine from localStorage.
 * Returns [] on SecurityError (private browsing) or missing key.
 *
 * @param {string} routineId ‚Äî e.g. "day2"
 * @returns {string[]}
 */
function getRecentIds(routineId) {
  try {
    const raw = localStorage.getItem(`gymup_recent_${routineId}`);
    return raw ? JSON.parse(raw) : [];
  } catch {
    return [];
  }
}

/**
 * Write the exercise IDs from a completed build to localStorage.
 * Silently no-ops on SecurityError or when localStorage is unavailable.
 *
 * @param {string} routineId ‚Äî e.g. "day2"
 * @param {string[]} ids ‚Äî exercise IDs from the most recent build
 */
function storeRecentIds(routineId, ids) {
  try {
    localStorage.setItem(`gymup_recent_${routineId}`, JSON.stringify(ids));
  } catch {
    // SecurityError ‚Äî private browsing or storage full. Ignore silently.
  }
}
```

**Step 3 ‚Äî Update each workoutDays.build() method to apply recency filter and store results**

Replace each `build()` method body with the localStorage-aware version. The pattern is:
1. Call `getRecentIds(this.id)` to get the array of recently-shown exercise IDs
2. Define `filterPool(pool)`: filter out exercises whose ID is in recentIds. If filtered pool has fewer items than needed, fall back to the full pool (prevents deadlock).
3. Call `pickRandom(filterPool(pool), count)` instead of `pickRandom(pool, count)` for each group
4. Assemble the result array
5. Call `storeRecentIds(this.id, result.map(e => e.id))` before returning

The PICK_COUNT (minimum exercises needed from a pool before fallback) per group per day:
- day1: biceps=1, triceps=1, shoulders=1, abs=3
- day2: chest=3, abs=3
- day3: back=3, abs=3
- day4: legs=3

**Part A ‚Äî Add renderCard() helper**

Add this private helper function immediately before `renderHome()` in app.js:

```javascript
/**
 * Renders a single exercise card as an HTML string.
 * SVG is injected via template literal ‚Äî safe because getSvg() uses only
 * controlled internal data (exercise.svgKey comes from exercises.js, not user input).
 *
 * @param {Object} exercise ‚Äî exercise object with _group added by build()
 * @returns {string} ‚Äî HTML string for one .exercise-card
 */
function renderCard(exercise) {
  const svgMarkup = getSvg(exercise.svgKey, '#82CDD8');
  const videoUrl  = `https://youtu.be/${escapeHtml(exercise.videoId)}`;
  const thumbUrl  = `https://img.youtube.com/vi/${escapeHtml(exercise.videoId)}/hqdefault.jpg`;
  const group     = escapeHtml((exercise._group || exercise.group).toUpperCase());

  return `
    <div class="exercise-card">
      <div class="card-svg-container">
        ${svgMarkup}
      </div>
      <div class="card-body">
        <span class="exercise-group-tag">${group}</span>
        <h2 class="card-title">${escapeHtml(exercise.name)}</h2>
        <p class="card-description">${escapeHtml(exercise.description)}</p>
        ${exercise.videoId ? `
        <a
          class="youtube-btn"
          href="${videoUrl}"
          target="_blank"
          rel="noopener noreferrer"
          aria-label="Watch ${escapeHtml(exercise.name)} on YouTube"
        >
          <img
            class="youtube-btn__thumbnail"
            src="${thumbUrl}"
            alt=""
            loading="lazy"
            width="320"
            height="180"
          />
          <span class="youtube-btn__label">Watch video</span>
        </a>` : ''}
      </div>
    </div>
  `;
}
```

Note: The SVG string from getSvg() is inserted directly into the template literal (not via escapeHtml) because it is a valid SVG markup string, not user-supplied text. This is safe ‚Äî svgKey values come from exercises.js (controlled data), and accentColor is hardcoded '#82CDD8'.
