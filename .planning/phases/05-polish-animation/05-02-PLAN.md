  <action>
Run a performance audit of index.html using one of these two methods (use whichever succeeds):

**Method A — Node.js Lighthouse CLI (preferred):**
```bash
npx lighthouse "file:///$(pwd)/index.html" \
  --throttling-method=simulate \
  --throttling.downloadThroughputKbps=1638 \
  --throttling.uploadThroughputKbps=768 \
  --throttling.rttMs=150 \
  --form-factor=mobile \
  --output=json \
  --output-path=/tmp/lighthouse-report.json \
  --chrome-flags="--headless" \
  --quiet 2>&1 | tail -5
```

Then extract key metrics:
```bash
node -e "
const r = JSON.parse(require('fs').readFileSync('/tmp/lighthouse-report.json','utf8'));
const a = r.audits;
console.log('FCP:', a['first-contentful-paint']?.displayValue);
console.log('TTI:', a['interactive']?.displayValue);
console.log('TBT:', a['total-blocking-time']?.displayValue);
console.log('Performance score:', r.categories?.performance?.score * 100);
"
```

**Method B — Manual Chrome DevTools audit (fallback if CLI fails):**
1. Open index.html in Chrome (file:// URL)
2. Open DevTools > Network tab > throttle to "Fast 4G"
3. Hard-reload (Cmd+Shift+R) and record:
   - DOMContentLoaded time (bottom status bar)
   - Load time (bottom status bar)
4. Open DevTools > Lighthouse tab
5. Set: Mobile device, simulated throttling, Performance only
6. Click "Analyze page load"
7. Record: FCP, TTI, TBT, Performance score

After obtaining results (by either method), write PERF-REPORT.md with the following structure:
