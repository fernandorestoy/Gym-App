  <action>
Two changes to js/app.js:

**Change 1 — Wrap render() with View Transitions:**

Replace the current `render()` function body (lines 271-276) with:

```javascript
function render() {
  const doRender = () => {
    document.getElementById('app').innerHTML =
      currentScreen === 'home'
        ? renderHome()
        : renderExercise(currentRoutine);
    applyCardStagger();
  };

  if (document.startViewTransition) {
    document.startViewTransition(doRender);
  } else {
    doRender();
  }
}
```

**Change 2 — Add applyCardStagger() helper:**

Add the following function IMMEDIATELY BEFORE the `render()` function (so it is defined before being called):

```javascript
/**
 * Apply staggered animation-delay to each .exercise-card after a render.
 * Each card gets an additional 80ms delay per index position.
 * Cards inside .exercise-pair inherit the card index based on DOM order,
 * not pair order — so all 6 cards stagger individually.
 */
function applyCardStagger() {
  document.querySelectorAll('.exercise-card').forEach((card, index) => {
    card.style.animationDelay = `${index * 80}ms`;
  });
}
```

**Important — regenerate focus logic:**

The existing regen handler (inside the event delegation block) does:
```javascript
render();
document.querySelector('[data-action="regenerate"]')?.focus();
```

The focus call runs SYNCHRONOUSLY after `render()`. With View Transitions, `render()` now returns before the transition completes. The `.regenerate-btn` element is inserted by `doRender()` inside the transition callback, so it exists in the DOM immediately (View Transitions renders synchronously then animates). The focus call is safe as-is.

Do NOT change the event delegation block. Only modify the `render()` function body and add the `applyCardStagger` function before it.
