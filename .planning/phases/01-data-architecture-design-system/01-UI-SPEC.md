## Spacing Scale

Declared values (multiples of 4):

| Token | Value | CSS Variable | Usage |
|-------|-------|-------------|-------|
| xs | 4px | `--space-xs: 4px` | Icon gaps, inline padding |
| sm | 8px | `--space-sm: 8px` | Compact element spacing |
| md | 16px | `--space-md: 16px` | Default element spacing, card inner padding base |
| lg | 24px | `--space-lg: 24px` | Card inner padding (required for gym legibility) |
| xl | 32px | `--space-xl: 32px` | Section padding, screen-level vertical rhythm |
| 2xl | 48px | `--space-2xl: 48px` | Major section breaks; also minimum touch target height |
| 3xl | 64px | `--space-3xl: 64px` | Page-level spacing, screen top padding |

Exceptions:
- Touch targets: 48px minimum height on all interactive elements (DSNG-06; NNGroup gym-context recommendation)
- Card inner padding: 24px (lg) on all sides — larger than default to ensure readability in bright gym light

**Source:** ARCHITECTURE.md (touch target specification), phase_context defaults

## Typography

| Role | Size | Weight | Line Height | Font | CSS Variable |
|------|------|--------|-------------|------|-------------|
| Body | 16px | 400 | 1.6 | Inter | `--font-body: 'Inter', sans-serif` |
| Label / Tag | 12px | 600 | 1.2 | Inter | (same family, uppercase) |
| Heading (exercise name) | 20px | 300 | 1.2 | Crimson Pro | `--font-heading: 'Crimson Pro', serif` |
| Display (app title / routine title) | 28px | 300 | 1.1 | Crimson Pro | (same family) |

Notes:
- Body minimum 16px — 15px is floor per ARCHITECTURE.md; 16px chosen for gym bright-light legibility
- Crimson Pro at weight 300 per REQUIREMENTS.md (DSNG-04). The current `index.html` loads Inter at wght 400;600;700;800 only — Crimson Pro import is entirely absent. Phase 1 replaces this.
- Do NOT use Teal or Dusty Rose as text color on any background. Navy on Cream is the only body text pairing (contrast ~9.4:1, WCAG AAA). Source: ARCHITECTURE.md contrast table.
- Labels/tags: Inter, weight 600, uppercase, 12px, used for muscle group pills (Sage Green background)

## Color

| Role | Hex | CSS Variable | Usage |
|------|-----|-------------|-------|
| Dominant (60%) | `#FFFEF1` | `--color-cream` | Page background (`body`), card backgrounds — NEVER pure white |
| Secondary (30%) | `#1A2742` | `--color-navy` | All headings, all body text, back button border |
| Accent (10%) | `#82CDD8` | `--color-teal` | Card borders (1–2px solid), interactive focus rings, muscle group tag background |
| Semantic — secondary accent | `#E58C8A` | `--color-dusty-rose` | Regenerate button background (reserved — do not use elsewhere in Phase 1) |
| Semantic — tertiary accent | `#5BA08A` | `--color-sage` | Muscle group pill/tag labels (large text only, ≥18px bold; contrast 3.6:1 AA large text only) |

Accent (`--color-teal`) reserved for:
1. Card border (1–2px solid `var(--color-teal)`)
2. Focus-visible outline on all interactive elements (3px solid `var(--color-teal)`, offset 3px)
3. Muscle group pill/tag background at low opacity (10–15% alpha)

Dusty Rose (`--color-dusty-rose`) reserved for:
- Regenerate button only (Phase 3 concern; declared here so token is available in `:root` from Phase 1)

Destructive: none in Phase 1. No destructive actions exist in this phase. Token declared as `--color-destructive: #E58C8A` for future use.

Color rules:
- Do NOT use `--color-teal` as body text on `--color-cream` — contrast is ~2.7:1, WCAG fail
- Do NOT use `--color-dusty-rose` as body text on `--color-cream` — contrast ~3.0:1, WCAG fail
- Do NOT use `--color-sage` as body text smaller than 18px bold — contrast 3.6:1 passes AA large text only
- No gradients anywhere. No shadows anywhere. No 3D effects. (DSNG-05)

**Current codebase gap:** `css/styles.css` declares dark theme tokens (`--bg-primary: #0D0D0D`, `--bg-card: #1A1A2E`, etc.) and rainbow gradient on `.logo`. Phase 1 replaces ALL `:root` variables and removes the `gradientShift` keyframe and all gradient references.

**Source:** REQUIREMENTS.md (DSNG-01 through DSNG-03, DSNG-05), phase_context, ARCHITECTURE.md (contrast table)

## Exercise Data Schema Contract

`exercises.js` must export the following structure after Phase 1. Schema comment is mandatory.

```javascript
/**
 * Exercise Database — Antigravity
 *
 * Structure: grouped object keyed by muscle group
 * Groups: chest | back | biceps | triceps | shoulders | abs | legs
 *
 * Per-exercise fields:
 *   id          {string}  Stable kebab-case ID, prefixed by group, zero-padded. e.g. "chest-001"
 *                         Unique across the entire library. Never reassigned once set.
 *   name        {string}  Display name in English. Title case.
 *   group       {string}  Must match the parent key exactly. e.g. "chest"
 *   description {string}  2–4 sentences. Action-focused how-to. No "you should" — direct imperative.
 *   videoId     {string}  YouTube video ID only (not full URL). e.g. "dQw4w9WgXcQ"
 *                         Thumbnail computed as: https://img.youtube.com/vi/${videoId}/hqdefault.jpg
 *                         Watch URL computed as: https://youtu.be/${videoId}
 *   svgKey      {string}  Key into svgIllustrations map. camelCase. e.g. "benchPress"
 */

export const exercises = {
  chest:     [ /* Exercise objects */ ],
  back:      [ /* Exercise objects */ ],
  biceps:    [ /* Exercise objects */ ],
  triceps:   [ /* Exercise objects */ ],
  shoulders: [ /* Exercise objects */ ],
  abs:       [ /* Exercise objects */ ],
  legs:      [ /* Exercise objects */ ],
};
```

Example exercise object (Phase 1 ships 1–3 placeholder entries per group to validate the schema; full content is Phase 4):

```javascript
{
  id: "chest-001",
  name: "Bench Press",
  group: "chest",
  description: "Lie flat on a bench with the bar over your lower chest. Lower the bar with control until it touches your chest, then press it back up to full extension. Keep your shoulder blades retracted throughout the movement.",
  videoId: "vcBig73ojpE",
  svgKey: "benchPress",
}
```

ID rules:
- Format: `{group}-{NNN}` (zero-padded to 3 digits)
- Assigned sequentially and never reused
- Group prefix matches the parent key exactly: `chest-001`, not `pecho-001` or `ch-001`

**Current codebase gap:** `exercises.js` uses Spanish group keys (`gluteos-piernas`, `brazos`, `pecho`, etc.) and Spanish names/descriptions. Phase 1 migrates to English keys (`chest`, `back`, `biceps`, etc.) as specified in REQUIREMENTS.md (DATA-01, DATA-02, DATA-04) and project context.

**Source:** REQUIREMENTS.md (DATA-01, DATA-02, DATA-04), ARCHITECTURE.md (data model section), SUMMARY.md

## Phase 1 Success Verification Checklist

These are the observable states that confirm Phase 1 is complete (derived from ROADMAP.md success criteria):

1. Open `index.html` directly in a browser (file:// — no server). Background is `#FFFEF1`. No pure white, no dark background.
2. Browser DevTools → Network tab shows `fonts.googleapis.com` loaded. Both Crimson Pro and Inter appear in Computed Styles for `body` and heading elements.
3. `exercises.js` exports a named `exercises` object with exactly 7 keys: `chest`, `back`, `biceps`, `triceps`, `shoulders`, `abs`, `legs`.
4. Each exercise entry has all 6 required fields: `id`, `name`, `group`, `description`, `videoId`, `svgKey`.
5. All exercise `id` values follow `{group}-NNN` pattern. No duplicates (verify with `Object.values(exercises).flat().map(e => e.id)` in browser console).
6. DevTools → Elements → `:root` computed styles show all `--color-*`, `--font-*`, `--space-*`, `--radius-*`, and `--transition-*` variables present.
7. No `box-shadow`, `linear-gradient`, or `radial-gradient` appears anywhere in computed styles.
