<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| exercises.js → app.js | Internal ES module import — same origin, file:// context |
| videoId → YouTube URL | At render time (Phase 3), videoId is concatenated into URLs — this is the boundary where untrusted data could matter |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | exercises.js data | accept | Static JS file on local filesystem. Single user, no network exposure in Phase 1. No injection vector. |
| T-02-02 | Information Disclosure | videoId field | accept | YouTube IDs are public identifiers for public videos. No PII. No secrets stored in exercises.js. |
| T-02-03 | Tampering | videoId concatenation (Phase 3) | mitigate | At Phase 3 render time: videoId values must only be used in href and img src attributes, never in innerHTML or eval(). This is a pre-mitigation note — enforcement is a Phase 3 task. |
</threat_model>
