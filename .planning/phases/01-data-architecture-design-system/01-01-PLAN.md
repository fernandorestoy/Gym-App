<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| file:// → browser | Static files opened directly — no server, no network trust boundary for app logic |
| index.html → Google Fonts CDN | External resource load; user's browser fetches font CSS from fonts.googleapis.com |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Information Disclosure | Google Fonts CDN request | accept | Personal tool, single user, no PII in font request. Font URL contains only family names — no user data. Acceptable for personal gym app. |
| T-01-02 | Denial of Service | Google Fonts unavailable | accept | `display=swap` ensures system fallback fonts (Georgia, -apple-system) render immediately if CDN is down. App remains usable at the gym. |
| T-01-03 | Tampering | No server — file:// only | accept | No server means no injection vector for Phase 1. Phase 1 has zero dynamic content. |
</threat_model>
